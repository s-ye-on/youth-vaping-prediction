## 담배 사용 데이터 전처리 
```text
TC_EC_MN의 9999 값은 이용지침서상 분기형 문항의 비해당 코드로 해석할 수 있으며,
 실제 원자료에서 TC_EC_LT와의 교차표를 확인한 결과 액상형 전자담배 평생 사용 경험이 없는 응답자에게만 나타났다.
따라서 본 연구에서는 TC_EC_MN=9999를 현재 액상형 전자담배 비사용자(y=0)로 분류하였다.
```

```text
본 연구의 타깃은 액상형 전자담배의 평생 사용 경험 여부가 아니라 현재 사용 여부이다.
따라서 평생 사용 경험이 있더라도 최근 30일 동안 사용하지 않은 응답자(TC_EC_MN=1)는 현재 비사용자(y=0)로 분류하였다.
반면 최근 30일 동안 1일 이상 사용한 응답자(TC_EC_MN=2~7)는 현재 사용자(y=1)로 분류하였다.
TC_EC_MN=9999는 이용지침서상 분기형 문항의 비해당 코드이며,
실제 교차표에서도 평생 사용 경험이 없는 응답자에게만 나타났으므로 현재 비사용자(y=0)로 처리하였다.
```

```text
본 연구는 현재 액상형 전자담배 사용 여부를 예측 대상으로 삼았기 때문에, 평생 사용 경험은 있으나 최근 30일 동안 사용하지 않은 과거 경험자는 현재 비사용자로 분류하였습니다.
향후 연구에서는 평생 미경험자, 과거 경험자, 현재 사용자를 구분하는 다중분류 모델을 통해 전자담배 사용 단계별 예측 요인을 비교할 수 있습니다. 
```

```text
일반담배 평생 경험을 넣자는 의견:
찬성. 해석 근거도 좋음.
다만 “원인”이 아니라 “니코틴 제품 경험과 흡연행동 노출을 반영하는 위험행동 지표”라고 표현.

음주 변수 포함:
찬성. 위험행동 군집을 반영하는 변수로 적절함.

가정 내 간접흡연 포함:
찬성. 가정 내 흡연환경 노출을 나타내는 환경 요인으로 적절함.

가구조사 미동의를 별도 범주로 두자는 의견:
찬성.

가구조사 미동의를 “문제가 있는 가정”으로 판단하자는 의견:
반대. 너무 강한 해석이고 낙인 위험이 있음.
```

```text
본 연구는 청소년의 액상형 전자담배 현재사용 여부를 예측하는 것을 목적으로 하며,
음주, 일반담배 경험, 간접흡연, 정신건강, 생활습관, 가정환경 변수는
현재사용 집단의 특성을 구분하는 예측 요인으로 사용하였다.
다만 본 자료는 횡단면 설문자료이므로, 해당 변수들이 액상형 전자담배 사용의 원인이라고 단정하지 않는다.
```

## 가구조사 미동의자 처리

```text
1단계: A_FM, 즉 가구조사 동의 여부 변수는 반드시 살린다.
2단계: 미동의자의 가족구성원/부모동거/부모학력/부모국적 변수는
       억지로 추정하지 않고 “가구조사 미동의”라는 별도 범주로 둔다.
```

미동의자들은 별도 범주를 만들어야 함  

미동의자는 8888이고, 부모 존재 여부는 동의자에 한해서 9999값들은 미존재 
가족 구성원 표시 
1. 아버지
2. 새아버지
3. 어머니
4. 새어머니
5. 할아버지 
6. 할머니
7. 형,누나,오빠,언니
8. 남동생 또는 여동생
9. 가족 구성원이 없음 
8888 비해당(가구조사 미동의자)

### 보고서 

```text
가족구성원 문항은 다중응답 문항이므로 각 가족 구성원 항목을 독립적인 이진 변수로 변환하였다.
 가구조사 미동의자는 가족구성원 정보가 관측되지 않는 별도 상태로 보아 family_info_private 변수를 1로 설정하고,
  개별 가족구성원 이진 변수는 0으로 처리하였다. 따라서 해당 0은 family_info_private와 함께 해석하였다.
```

### 주관적 건강 인지 `PR_HT` 
```text
1 = 매우 건강한 편이다
2 = 건강한 편이다
3 = 보통이다
4 = 건강하지 못한 편이다
5 = 매우 건강하지 못한 편이다
```

### 주관적 체형 인지 `PR_BI`
```text
1 = 매우 마른 편이다
2 = 약간 마른 편이다
3 = 보통이다
4 = 약간 살찐 편이다
5 = 매우 살찐 편이다
```

#### 정리 
```text
PR_HT 주관적 건강인지
→ 그대로 1~5 사용 가능
→ 단, 컬럼명을 subjective_unhealthy_level로 바꿔서 방향성 명확화

PR_BI 주관적 체형인지
→ 1~5 그대로 두기보다 one-hot 추천
→ 체형 인식은 선형 관계를 가정하기 애매하기 때문

스마트폰 사용 시간
→ INT_SPWD_TM + INT_SPWK_TM 합산 추천
→ 둘 다 분 단위라면 smartphone_total_time으로 만드는 게 맞음
```
